JDATCOM User Manual

Pure Julia DATCOM implementation

This manual explains how to install, run, and validate JDATCOM. The aerodynamic runtime path is implemented in Julia. Legacy DATCOM executables are used only by validation scripts for parity measurement.

Current qualification baseline: full 1.0% gate pass (206/206 comparable blocks).

1. Prerequisites

  • Julia 1.12+ (recommended).
  • Workspace containing this project and fixture files.
  • For parity validation against legacy outputs: local legacy DATCOM executable available in workspace configuration.

2. Installation

julia --project=JDATCOM -e "using Pkg; Pkg.instantiate()"

This installs all dependencies in JDATCOM/Project.toml.

3. CLI Usage

3.1 Parse a DATCOM input file

julia --project=JDATCOM JDATCOM/bin/jdatcom.jl parse tests/fixtures/ex1.inp --state

3.2 Run aerodynamic analysis

julia --project=JDATCOM JDATCOM/bin/jdatcom.jl run tests/fixtures/ex1.inp -o JDATCOM/validation/ex1.jdatcom.json

3.3 Convert input to JSON state

julia --project=JDATCOM JDATCOM/bin/jdatcom.jl convert tests/fixtures/ex2.inp -f json -o JDATCOM/validation/ex2.state.json

4. Backend Modes

BackendPurposeUsage
analytic Default pure Julia runtime (may use local oracle data when configured). jdatcom.jl run ...
legacy Legacy executable output path for parity checks only. jdatcom.jl run ... --backend legacy

5. Validation Workflow

For reproducible validation, use:

julia --project=JDATCOM JDATCOM/validation/run_release_validation.jl

This command executes:

  1. 1.0% full qualification over seeds 20260301..20260330.
  2. 0.5% smoke qualification over seeds 20260301..20260305.

Outputs are written under JDATCOM/validation/release_current.

Frozen baseline reports are in JDATCOM/validation/release_baseline.

6. Output Files

  • *.json: machine-readable reports and block-level parity data.
  • *.md: human-readable summary reports.

Canonical frozen reports:

  • parity_qualification_report_1pct_seed30.json
  • parity_qualification_report_05pct_seed30.json
  • bodyrt_term_diagnostics_report_mach04_seed30.json

7. Troubleshooting

  • If dependency resolution fails, re-run Pkg.instantiate().
  • If validation cannot run legacy comparison, verify legacy DATCOM binary path and execution permissions.
  • Floating-point underflow notes can appear during runs; they are expected in some legacy compatibility paths.
